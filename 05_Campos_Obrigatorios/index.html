Mapa dos Campos do Ticket

Resumo unico dos 4 grupos oficiais para leitura rapida da regra operacional.

Obrigatorio

Obrigatorios principais

Base minima para abrir e qualificar o ticket sem retrabalho.

Ticket - abertura

  • Nome do solicitante/cliente
  • E-mail
  • Telefone
  • Solicitante
  • Tipo de atendimento
  • Categoria
  • Produto
  • Marca do produto
  • Numero de serie do equipamento, com excecao contextual
  • Assunto
  • Descricao

Contato/Cliente

  • Nome
  • Sobrenome
  • Nome da conta
  • E-mail
  • Telefone/Celular
Condicional

Condicionais

Obrigatorio somente quando o contexto regional ou de solicitante exige.

  • Estado: obrigatorio apenas Brasil
  • Provincia: obrigatoria apenas Argentina
  • Nome da assistencia/distribuidor: obrigatorio para Assistencia / Distribuidor
Automatico

Automaticos/sistemicos

Dados preenchidos e atualizados pelo sistema ao longo da jornada.

  • ID do ticket
  • Data de abertura
  • Canal
  • Proprietario do ticket
  • Status
  • Prioridade, calculada pela matriz
  • SLA de primeira resposta
  • SLA de resolucao
  • Tempos e indicadores de SLA
Desejavel

Desejaveis oficiais

Nao bloqueiam abertura, mas elevam a qualidade da leitura operacional.

  • Pais
  • Zero hora / Out of the box
  • Necessita peca
  • Parceiro/distribuidor, quando nao obrigatorio
  • RAC Enviada? - Sim/Nao
  • Tipo de assistencia tecnica
  • Motivo do fechamento
  • Solucao em campo/remoto
  • Peca utilizada
  • Quantidade final de pecas

Nao fazem parte da lista oficial de desejaveis nesta fase: acoes executadas, quantidade de pecas, peca solicitada, codigo da peca, datas/status da peca, numero do pedido, causa provavel e data prevista de retorno ao cliente.

Impacto direto em

  • Qualidade de dados
  • Fluxo
  • Prioridade
  • SLA
  • KPI
  • Auditoria
Consultar Matriz Consolidada

Regras criticas

  • Numero de serie e obrigatorio, com excecao contextual.
  • Estado so Brasil.
  • Provincia so Argentina.
  • Nome da assistencia/distribuidor so quando aplicavel.
  • Prioridade e obrigatoria e calculada pela matriz.
  • Zero hora / Out of the box e desejavel, mas impacta prioridade.

Como usar este modulo

  • Use a home para entender a regra.
  • Use a Matriz Consolidada para consultar o detalhe completo.
  • Coordenadores devem usar a matriz para cobrar completude.
  • Agentes devem usar como referencia para preenchimento correto.
